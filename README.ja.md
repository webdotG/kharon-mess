## これは何ですか？

Kharonは小規模チーム向けのミニマリスト暗号化メッセンジャーです。メッセージは2分間生存した後、永久に消えます。アカウントなし、電話番号なし、メッセージ履歴なし、自分のサーバー以外は使用しません。

ギリシャ神話の冥界の渡し守、カロンにちなんで命名されました。川を渡ったものは戻らない。

## 設計思想

ほとんどのメッセンジャーはメッセージを他人のサーバーに保存し、あなたが誰かを知り、データを第三者に提出するよう強制される可能性があります。Kharonはまったく逆の原則で構築されています：

- サーバーは**単純なリレー** — 暗号化されたブロブを転送するだけで、それ以外は何も知らない
- **あなたがサーバーを運営** — 自分のハードウェアで
- メッセージは**2分後に自己消滅** — 設計上、履歴なし
- **アカウントなし** — あなたのアイデンティティは暗号鍵ペア、それだけ
- 連絡先の追加 = 公開鍵を直接交換、オフラインまたはコピー＆ペースト

## メッセージの流れ

```
[あなたのスマートフォン]
  1. メッセージを入力
  2. 受信者の公開鍵で暗号化（X25519 + XSalsa20-Poly1305）
  3. ランダムな24バイトのノンスを暗号文の前に追加
  4. WSS経由でbase64ブロブを送信

       ↓  TLS 1.3  ↓

[Nginx]
  5. TLS終端、レート制限、WebSocket検証

[Node.jsリレー]
  6. 暗号化ブロブを受信 — 平文は絶対に見ない
  7. 受信者のWebSocket接続にルーティング
  8. オフラインの場合：RAMキューに保存（最大2分TTL）

       ↓  TLS 1.3  ↓

[受信者のスマートフォン]
  9. ブロブを受信、自分の秘密鍵で復号
  10. 平文を表示 — RAMにのみ保存、ディスクには書き込まれない
```

## セキュリティアーキテクチャ

### 暗号化

| 用途 | アルゴリズム | 理由 |
|---|---|---|
| 鍵交換 | X25519（Curve25519 DH） | 高速、安全、広く監査済み |
| 暗号化 | XSalsa20-Poly1305 | 認証暗号化 — 改ざん検出 |
| ノンス | メッセージごとにランダム24バイト | 192ビットエントロピー、衝突確率≈0 |
| 鍵の保存 | Android Keystore + EncryptedSharedPreferences | ハードウェア保護、抽出不可 |
| 連絡先の保存 | Room + SQLCipher | 静止時暗号化データベース |

### サーバーが知っていること

| データ | サーバーは見える？ |
|---|---|
| メッセージ内容 | ❌ 絶対に — 暗号化ブロブのみ |
| 送信者の身元 | ❌ 公開鍵のみ（44文字のbase64） |
| メッセージ履歴 | ❌ TTL 2分、その後RAMから削除 |
| あなたの名前 | ❌ 登録不要 |
| あなたの電話番号 | ❌ 不要 |
| あなたのIPアドレス | ⚠️ はい、接続ログに |

### 防御層

**サーバー:** TLS 1.3のみ · レート制限 · 接続制限 · helloタイムアウト · トークンバケット · MAX_CLIENTS上限 · ペイロード検証

**クライアント:** WSSのみ · 証明書ピンニング · 指数バックオフ · FLAG_SECURE · バックアップ無効 · rootチェック · リプレイ攻撃の重複排除

---

## Kharonが保護しないもの

- **物理的なデバイスの押収** — 現在のセッションはRAMに保存；過去のセッションはすでに消えている
- **サーバー運営者の侵害** — 接続メタデータは見えるが、メッセージ内容は見えない
- **rootされたデバイス** — Android Keystoreが回避される可能性がある
- **トラフィック分析** — 観察者はあなたがサーバーに接続していることは見えるが、内容は見えない
- **エンドポイントマルウェア** — スクリーンキャプチャマルウェアからメッセンジャーは保護できない

## サーバーの展開

### 要件
- Docker + docker-compose
- ドメインまたは静的IP
- TLS証明書（Let's Encrypt）

### クイックスタート

```bash
git clone https://github.com/webdotG/kharon-mess
cd kharon-mess/server

cp .env.example .env

# TLS証明書の取得
certbot certonly --standalone -d yourdomain.com
mkdir certs
cp /etc/letsencrypt/live/yourdomain.com/fullchain.pem certs/
cp /etc/letsencrypt/live/yourdomain.com/privkey.pem certs/

docker compose up -d
```

### 環境変数

| 変数 | デフォルト | 説明 |
|---|---|---|
| `MSG_TTL_MS` | `120000` | メッセージの生存時間（ミリ秒） |
| `MAX_MSG_SIZE` | `4096` | 最大ペイロードサイズ（バイト） |
| `MAX_CONNS_PER_IP` | `3` | IPあたりの最大同時接続数 |
| `MAX_CLIENTS` | `100` | 最大接続ユーザー数 |
| `MSG_RATE_LIMIT` | `3` | 接続あたりの毎秒メッセージ数 |

## なぜ「Kharon」？

カロン（Χάρων）はハデスの渡し守で、スティクス川とアケロン川を渡って魂を運んでいました。与えられたものを受け取り、向こう側に運ぶ — それで終わりです。記録なし。帰り道なし。

これがこのメッセンジャーのすることです。
