## ما هذا؟

خارون هو تطبيق مراسلة مشفّر بسيط للفرق الصغيرة. تعيش الرسائل لمدة دقيقتين ثم تختفي إلى الأبد. لا حسابات، لا أرقام هواتف، لا سجل رسائل، لا خوادم لا تملكها.

سُمّي على اسم خارون — ملّاح العالم السفلي في الميثولوجيا الإغريقية. ما يعبر النهر لا يعود.

## الفلسفة

معظم تطبيقات المراسلة تخزّن رسائلك على خادم شخص آخر، وتعرف من أنت، ويمكن إجبارها على تسليم بياناتك. خارون مبني على المبدأ المعاكس تماماً:

- الخادم **مجرد وسيط** — يُمرّر الكتل المشفّرة ولا يعرف شيئاً آخر
- **أنت تُشغّل الخادم** — على جهازك الخاص
- الرسائل **تدمّر نفسها** خلال دقيقتين — لا سجل بالتصميم
- **لا حسابات** — هويتك هي زوج مفاتيح تشفير، لا أكثر
- إضافة جهة اتصال = تبادل المفاتيح العامة مباشرةً، دون إنترنت أو عبر نسخ ولصق

## كيف تنتقل الرسالة

```
[هاتفك]
  ١. تكتب الرسالة
  ٢. تشفيرها بالمفتاح العام للمستلم (X25519 + XSalsa20-Poly1305)
  ٣. إضافة 24 بايت عشوائي في البداية
  ٤. إرسال كتلة base64 عبر WSS

       ↓  TLS 1.3  ↓

[Nginx]
  ٥. إنهاء TLS، تحديد المعدل، التحقق من WebSocket

[Node.js المرحّل]
  ٦. يستقبل الكتلة المشفّرة — لا يرى النص الصريح أبداً
  ٧. يُوجّه إلى اتصال WebSocket الخاص بالمستلم
  ٨. إذا كان غير متصل: يُخزّن في قائمة انتظار الذاكرة (مدة أقصاها دقيقتان)

       ↓  TLS 1.3  ↓

[هاتف المستلم]
  ٩. يستقبل الكتلة، يفكّها بمفتاحه السري
  ١٠. يعرض النص — محفوظ في الذاكرة فقط، لا يُكتب على القرص أبداً
```

## بنية الأمان

### التشفير

| الاستخدام | الخوارزمية | السبب |
|---|---|---|
| تبادل المفاتيح | X25519 (Curve25519 DH) | سريع، آمن، مُدقَّق على نطاق واسع |
| التشفير | XSalsa20-Poly1305 | تشفير موثّق — يكشف العبث |
| الرقم العشوائي | 24 بايت عشوائي لكل رسالة | انتروبيا 192 بت، احتمال التصادم ≈ صفر |
| تخزين المفاتيح | Android Keystore + EncryptedSharedPreferences | محمي بالأجهزة، لا يمكن استخراجه |
| تخزين جهات الاتصال | Room + SQLCipher | قاعدة بيانات مشفّرة |

### ماذا يعرف الخادم

| البيانات | الخادم يرى؟ |
|---|---|
| محتوى الرسالة | ❌ أبداً — كتلة مشفّرة فقط |
| هوية المُرسل | ❌ مفتاح عام فقط (44 حرف base64) |
| سجل الرسائل | ❌ TTL دقيقتان، ثم يُحذف من الذاكرة |
| اسمك | ❌ لا تسجيل |
| رقم هاتفك | ❌ غير مطلوب |
| عنوان IP | ⚠️ نعم، في سجلات الاتصال |

### طبقات الحماية

**الخادم:** TLS 1.3 فقط · تحديد المعدل · حدود الاتصال · مهلة hello · دلو الرموز · حد MAX_CLIENTS · التحقق من الحمولة

**العميل:** WSS فقط · تثبيت الشهادة · تراجع أسي · FLAG_SECURE · تعطيل النسخ الاحتياطي · كشف root · إزالة ازدواجية هجمات الإعادة

## ما لا يحميك منه خارون

- **مصادرة الجهاز المادي** — الجلسة الحالية في الذاكرة؛ الجلسات السابقة اختفت بالفعل
- **مشغّل خادم مخترق** — يمكنه رؤية بيانات الاتصال لكن ليس محتوى الرسائل
- **جهاز مكسور (rooted)** — يمكن تجاوز Android Keystore
- **تحليل حركة البيانات** — يمكن للمراقب رؤية اتصالك بالخادم لكن ليس ما تقوله
- **برامج ضارة في الجهاز** — لا يوجد تطبيق مراسلة يحمي من برامج التقاط الشاشة

## نشر الخادم

### المتطلبات
- Docker + docker-compose
- نطاق أو IP ثابت
- شهادة TLS (Let's Encrypt)

### البدء السريع

```bash
git clone https://github.com/webdotG/kharon-mess
cd kharon-mess/server

cp .env.example .env

# الحصول على شهادة TLS
certbot certonly --standalone -d yourdomain.com
mkdir certs
cp /etc/letsencrypt/live/yourdomain.com/fullchain.pem certs/
cp /etc/letsencrypt/live/yourdomain.com/privkey.pem certs/

docker compose up -d
```

### متغيرات البيئة

| المتغير | الافتراضي | الوصف |
|---|---|---|
| `MSG_TTL_MS` | `120000` | مدة حياة الرسالة بالميلي ثانية |
| `MAX_MSG_SIZE` | `4096` | الحجم الأقصى للحمولة بالبايت |
| `MAX_CONNS_PER_IP` | `3` | أقصى اتصالات متزامنة لكل IP |
| `MAX_CLIENTS` | `100` | أقصى عدد مستخدمين متصلين |
| `MSG_RATE_LIMIT` | `3` | رسائل في الثانية لكل اتصال |
